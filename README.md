# music-hit-tracker

Телеграм-бот, який відстежує нову російську музику на YouTube і надсилає сповіщення про треки, що швидко набирають перегляди.

## Як працює

Скрипт `music_tracker.py` запускається за розкладом через GitHub Actions
(4 рази на день) і:

1. Збирає відео з YouTube — популярне в категорії «Музика» (RU) та результати
   пошуку за низкою запитів про новинки.
2. Фільтрує: лишає лише кириличні назви, відсіює україномовні та застарілі
   (старші за `MAX_AGE_DAYS`) відео.
3. Рахує перегляди на день (`views/day`) для кожного відео:
   - **≥ 50 000/день** — «щоденний хіт», сповіщення надсилається одразу
     (кожен трек — лише раз, дедуплікація через `notified_songs.json`);
   - **5 000–50 000/день** — кандидат у суботній дайджест
     (накопичується у `weekly_songs.json`).
4. По суботах (UTC) надсилає дайджест накопичених кандидатів і очищує список.

## Файли стану (комітяться воркфлоу автоматично)

| Файл | Призначення |
| --- | --- |
| `notified_songs.json` | ID уже надісланих хітів → дата (щоб не повторювати). Старі записи автоматично обрізаються через `NOTIFIED_RETENTION_DAYS`. |
| `weekly_songs.json` | Кандидати поточного тижня для суботнього дайджесту. |
| `last_notification.json` | Дати останнього щоденного та суботнього сповіщення. |

## Необхідні секрети репозиторію

Додайте в **Settings → Secrets and variables → Actions**:

| Секрет | Опис |
| --- | --- |
| `YOUTUBE_API_KEY` | Ключ YouTube Data API v3. |
| `TELEGRAM_BOT_TOKEN` | Токен бота від @BotFather. |
| `TELEGRAM_CHAT_ID` | ID чату/каналу, куди надсилати сповіщення. |

Воркфлоу пушить оновлений стан назад у репозиторій, тому потрібні права
`contents: write` (уже налаштовані у файлі воркфлоу).

## Локальний запуск

```bash
pip install requests
export YOUTUBE_API_KEY=...   TELEGRAM_BOT_TOKEN=...   TELEGRAM_CHAT_ID=...
python music_tracker.py
```

## Налаштування

Пороги та поведінку можна змінити константами на початку `music_tracker.py`
(`VIEWS_PER_DAY_THRESHOLD`, `WEEKLY_MIN_VIEWS_PER_DAY`, `MAX_AGE_DAYS`,
`SEARCH_QUERIES`, `REGIONS`).
